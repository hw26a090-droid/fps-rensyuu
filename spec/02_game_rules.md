# REFLEX LABORATORY ゲーム仕様書
## 02. ゲームフローとルール定義

本章では、ゲーム全体のライフサイクルおよび、プレイヤーの集中を阻害しないための画面遷移、そしてフライング操作に対するペナルティ仕様を定義します。

---

### 1. 各ゲームフェーズ（状態遷移）
ゲームは以下の4つの状態で遷移します。

```
 [IDLE (初期画面)]
        │ (テスト開始ボタン、または [Space] キー押下)
        ▼
 [WAITING (待機画面)]
        │ (1.5s 〜 4.0s のランダム待機)
        ├─────────────────────────────┐
        │ (クリック検知: フライング無し)   │ (クリック検知: フライング有り/お手つき)
        ▼                             ▼
 [ACTIVE (標的出現: CLICK!)]        [RESULT (ラウンド結果)]
        │                             │ (おてつきペナルティ: 1000ms)
        │ (クリック検知: 反応時間測定)  │
        └──────────────┬──────────────┘
                       ▼
                 [RESULT (ラウンド結果)]
                       │ (約1.5秒間のカウントダウン & プログレスバー表示)
                       ├ 5ラウンド未満 ──► [WAITING] へ戻り、次ラウンド
                       └ 5ラウンド完了 ──► [FINAL RESULT] へ
```

---

### 2. 状態ごとの画面仕様

#### ① IDLE (初期画面)
- **UI特徴**:
  - メインタイトル、ルールの3ステップ説明（1.待つ / 2.即クリック / 3.スコア算出）、フライング警告リボンを表示。
  - テストを即起動できる「テスト開始」ボタンの設置。
  - キーボードの `[Space]` キーによる開始バインディング。 

#### ② WAITING (待機画面)
- **UI特徴**:
  - **極限まで情報を削ぎ落とした「静かで無駄のない」暗い画面**。
  - 画面中央に赤色の小さなパルスランプと「WAIT...」のミニマルな文字だけを表示。
  - ターゲット予告やプレイヤーを混乱させるテキストは一切排除し、完全に標的の突発表示にリフレッシュ、集中できるよう誘導する。
- **処理**:
  - 遷移時、1500ミリ秒〜4000ミリ秒のランダムな待ち時間を設定しタイマーを始動。

#### ③ ACTIVE (標的出現)
- **UI特徴**:
  - **最大級の視認性**を持つ高輝度シアン色の特大サークル（CLICK!）が中央に出現。
  - 背景には波紋（Ping）アニメーションが広がり、視覚的な変化を脳に最速で伝達する。
  - 画面全体のどこをタップ/クリックしても反応する「特大クリック領域」となる。

#### ④ RESULT (ラウンド一時結果)
- **UI特徴**:
  - 当該ラウンドの反応速度（ms）を巨大な高輝度フォントで表示。
  - 次のラウンドへの移行状況を知らせるために、自動ローディングプログレスバーを配置（約1.5秒間で自動進行、クリック等による手動移動の必要を排除、リズムの維持）。

#### ⑤ FINAL RESULT (最終セッション結果)
- **UI特徴**:
  - 5ラウンド全ての計測完了時に遷移。
  - **5ラウンド平均反応速度（ミリ秒）**をハイライト表示。
  - オフセット補正後の実測ランク評価バッジ（最高、とても良い等）、および5回の詳細データリスト一覧。
  - 「履歴をリセット」ボタンへのアクセス（誤操作防止用の、はい・いいえクリックダイアログ制御）。

---

### 3. フライング（お手つき）時の厳密な制約
極限の反射速度を測るため、不正確な連打（予測クリック・ヤマ勘クリック）に対し、厳密かつゲーム性を保つためのルールを組み込みます。

- **検知**:
  - `WAITING`（待機中）の画面で、ターゲットが出現する前にクリックやタップ、`[Space]` 押下を検知した場合に「おてつき」と判定。
- **ペナルティルール**:
  - フライングをしたラウンドの反応速度は、強制的に **1000ms（1秒）** として処理する。
  - ターゲット出現画面には移行せず、即座に結果表示（おてつき警告・1000msペナルティ表示）へと遷移する。
  - 1000msがセッション平均スコアに組み入れられるため、プレイヤーは必然的に「ヤマ勘連打」を行わなくなります。
