# REFLEX LABORATORY仕様書 - 02. ゲームプレイフローと状態遷移

## 2.1 状態定義 (GameState)
ゲームは以下の4つの基本状態でループします。

| 状態名 | 定義 | 詳細 |
| :--- | :--- | :--- |
| **IDLE** | 待機状態 | 起動直後、または5ラウンド終了後の完全リセット状態。開始トリガーを待つ。 |
| **WAITING** | 測定前待機 | プレイヤーは画面中央の目印ドットを注視する待機状態（円は非表示）。 |
| **ACTIVE** | サークル点灯中 | ドットが消え、設定で選択された色（シアン・緑・ピンク・黄色・白）のサークルが中央に突然出現した状態。 |
| **RESULT** | 各回/最終結果表示 | 反応速度のミリ秒スコア、またはお手つき判定が表示されている状態。 |

---

## 2.2 1回のテスト（5ラウンド）のゲームループ

1. **セッション開始 (IDLE -> WAITING)**
   - 開始ボタンクリック、またはキーボードの**スペースキー押下**で第1ラウンドが開始。
   - WAITING状態へ移行する。
2. **ランダム待機 (WAITING)**
   - 待ち時間中は円を表示せず、画面中央に小さなドット（点）のみを表示します。プレイヤーはここに視線を集めて待機します。
   - 1.5秒から4.0秒のランダムな遅延時間が生成される。
3. **サークル点灯 (WAITING -> ACTIVE)**
   - 指定したランダム遅延時間経過後、`requestAnimationFrame` を用いて同期レンダリング。
   - 画面中央の目印用ドットが消えると同時に、設定で選択された色（シアン・緑・ピンク・黄色・白のいずれか）のサークルが中央に突然出現。
   - サークル内には「クリック！」と表示。
   - `performance.now()` でこのサークルが出現した瞬間のタイムスタンプ（ミリ秒）を取得し、`startTime` とする。
4. **入力検知によるタイミングの記録 (ACTIVE -> RESULT)**
   - `PointerDown` イベント（タッチやマウスボタン押し込み）を検知。
   - 検知した瞬間の `performance.now()` と `startTime` の差分を計算（ミリ秒、小数点第1位）。
   - `RESULT` 状態で、そのラウンドの数値（例: `158.4 ms`）を表示する。
5. **進捗管理と自動遷移**
   - **1〜4ラウンド終了時:** 結果表示（RESULT）の1.5秒後、自動的に次のラウンドの `WAITING` 状態へ移行する。
   - **5ラウンド終了時:** 5回すべてのスコアを収集し、合計から平均スコアを計算。最終評価（ランク）を表示し、履歴に記録する。

## 2.3 キーボード操作と「お手つき（フライングペナルティ）」の排他仕様
- **キーボード（スペースキー）の役割限定:**  
  スペースキーは、**「ゲームの開始（IDLE時）」**および**「ゲームの再挑戦（5ラウンド終了後のRESULT時）」**のショートカットキーとしての定義に限定します。
- **お手つき判定のトリガー:**  
  フライング（お手つき）およびACTIVE時の反応入力は、**「画面上での PointerDown（タッチ/クリック）」のみ**を対象とします。WAITING状態中にスペースキーを押下しても、フライング（お手つきペナルティ）としては判定しません（誤反応防止のため）。
- **お手つき（フライング）時のペナルティ処理:**  
  - WAITING状態（サークル点灯前）において、プレイヤーが画面を PointerDown した場合に判定されます。
  - 進行中のWAITINGタイマーを中断する。
  - **そのラウンドのスコアを「強制的に1000ms（ペナルティ時間）」として処理する**。
  - お手つき結果を画面に表示し、1.55秒後に自動的に次のラウンド等へ進む。
- **目的:**  
  あてずっぽうによるクリックを強力に防止し、純粋な反射神経を測定するためのゲームバランス上の措置。

## 2.4 音響フィードバック（Web Audio API）
反応速度測定の評価を損なうことなく、操作に対する十分な手応え（フィードバック）を提供するため、外部音声ファイルを使用しない**Web Audio APIによるデジタル合成音（シンセサイザー）**を実装しています。

1. **ターゲット出現音 (ピッ - 廃止)**
   - **理由:** 音声（聴覚刺激）が反応のきっかけになると、視覚反応速度の正確な測定を妨げる(聴覚反応時間は視覚反応時間より通常速いため)ため、高精度測定の信頼性維持を最優先し**廃止（無音化）**されました。

2. **測定成功音 (ポン)**
   - **発生条件:** ACTIVE状態中に画面がタップされ、反応時間を記録した瞬間。
   - **音響仕様:** 軽やかで心地よいマイルドな成功音（550Hzから400Hzへ急激にスイープするSine波、長さ約180ms、指数的音量減衰）。
   - **目的:** クリックが無事に受理されたことをユーザーへ心地よく瞬時にフィードバックします。

3. **お手つき/フライング警告音 (ブー)**
   - **発生条件:** WAITING（待機）状態中にフライングでタップしてしまった瞬間。
   - **音響仕様:** 低くザラザラした不協和警告音（140Hzおよび143Hzのデチューンされた鋸歯状波/Sawtooth波を2個重合、長さ約350ms、指数的音量減衰）。
   - **目的:** 視線固定中のフライングに対し直感的なペナルティ警告感覚を与えます。

※すべての音響は音量レベリングが徹底されており（控えめに制御）、測定・平均統計ロジックや `performance.now()` による高精度測定部分への阻害を一切行いません。また、UI上の効果音設定（オン/オフ）からすべての効果音を無効化（無音化）でき、その設定は `localStorage`（キー：`reaction_tester_soundenabled`）に保存され再起動後も自動で適用されます。
