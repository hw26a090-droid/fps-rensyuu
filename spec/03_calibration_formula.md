# REFLEX LABORATORY仕様書 - 03. キャリブレーションと判定評価

## 3.1 キャリブレーション（モニターリフレッシュレート補正）
プレイヤーが使用するモニターのリフレッシュレート（Hz数）によって、画面にサークルが表示されてから描画されるまでの「平均的な待ち時間」が物理的に変化します。
これを不公平なラグと捉え、本システムでは **240Hz モニター** を基準点としてオフセット補正をかけます。

### 補正係数の数式
$$offset = \begin{cases} 0 & (hz = 240) \\ \frac{500}{hz} - 2.1 & (hz \neq 240) \end{cases}$$

- **例1: 60Hz モニターの場合**  
  $\frac{500}{60} - 2.1 = 8.33 - 2.1 \approx 6.2 \text{ ms}$ のオフセットが、基準閾値に加算されます（閾値が優しくなる）。
- **例2: 144Hz モニターの場合**  
  $\frac{500}{144} - 2.1 = 3.47 - 2.1 \approx 1.4 \text{ ms}$ のオフセット。

## 3.2 リフレッシュレートの検知方法
1. **実測自動検知（rAFスニッフィング）**  
   `requestAnimationFrame` をループさせて1000ミリ秒間に描画されたフレーム数（frames）からHz数をリアルタイム計測します。
2. **手動切り替え（プルダウンメニュー）**  
   以下の規格リフレッシュレートから手動で選択可能です。
   - `60Hz`, `75Hz`, `90Hz`, `120Hz`, `144Hz`, `165Hz`, `240Hz`, `280Hz`, `360Hz`, `500Hz`
3. **ローカル保持**  
   設定されたHz値はローカルストレージ（キー：`reaction_tester_refreshrate`）に保存され、再訪問時に引き継がれます。

## 3.3 評価（ランク）判定基準一覧
平均スコア $T$ に対して、リフレッシュレート $hz$ に基づく $offset$ を加味した閾値で判定を行います。

| ランク名 | 判定条件 (単位: ms) | 240Hz基準値 |
| :--- | :--- | :--- |
| **最高** | $T \le 150 + offset$ | 150.0 ms 以下 |
| **とても良い**| $T \le 170 + offset$ | 150.1 〜 170.0 ms |
| **良い** | $T \le 190 + offset$ | 170.1 〜 190.0 ms |
| **平均以上** | $T \le 220 + offset$ | 190.1 〜 220.0 ms |
| **平均** | $T \le 260 + offset$ | 220.1 〜 260.0 ms |
| **平均以下** | $T \le 320 + offset$ | 260.1 〜 320.0 ms |
| **遅い** | $T \le 400 + offset$ | 320.1 〜 400.0 ms |
| **とても遅い**| $T \le 500 + offset$ | 400.1 〜 500.0 ms |
| **練習が必要**| $T > 500 + offset$ | 500.1 ms 超 |
